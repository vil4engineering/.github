# Contributing

**vil4engineering** is a long-term engineering organization for learning, building, and teaching across two generations.

Curriculum and campus docs live in [`univer`](https://github.com/vil4engineering/univer).

## Before you open a PR

1. Prefer evidence: code, lab, test, or documented decision — not opinion alone.
2. Keep changes small and reviewable.
3. Follow Conventional Commits: `type(scope): description`.
4. Do not invent experience or private career facts in public repos.
5. Documentation lives in the repo (`README.md`, `docs/`, `campus/`) — not in GitHub Wiki.

## Workflow

1. Open an issue or Discussion when the problem is unclear.
2. Create a branch from `main`.
3. Add or update tests / Evidence when behavior changes.
4. Fill the pull request template.
5. Wait for required checks (when configured).

## Audiences

Work should help at least one of:

- becoming a better engineer;
- helping the next generation become one.

## Language

- Public product / org README: English is fine.
- Curriculum prose in `univer`: Russian primary + English technical terms (see [`campus/LANGUAGE.md`](https://github.com/vil4engineering/univer/blob/main/campus/LANGUAGE.md)).
- Code identifiers and API names: English as-is.
